# How I got into this?
I originally didn't even pay for a AI subscription until Theo made t3.chat, which I bought for 8 dollars per month, when I have problems, I just throw it to Gemini 2.5 Pro to tell me how to fix it, and how can I learn from it. However, with that I slowly converted to a sorta vibe guy, some of my word documents are AI generated just because I'm too lazy to write it.

## Vibe coding
My first taste of (partly) vibe coding is with the Qwen Code 3 with Cline, I selected it because I'm a cheap person, I linked my openrouter API key, and wow, it just did what I wanted for me, with the issue of it changes random crap that I don't want the AI to modify. So I thought that is too annoying, and that made me hate the idea of vibe coding.

## I bought a Claude Pro subscription
At the start of 2026, I purchased a one month Claude Pro subscription, and I have been fallen into the Claude trap, Claude Code is awesome, however, I hit the limits already on the first 6 hours of me getting a sub, a tried to run two Claude Code instances, and another OpenCode instance using the Claude adapter.
## Oh the quality of the code is better than I thought...
After vibe coding for a project that I sloppy wrote a few months ago, it made it so much better (aka this website), the entirety of the new admin panel is written by Claude Opus 4.5 🫥, I felt, well, kinda useless after that discovery, I still write code, but the AI now can just do what I've want with maybe just one prompt...

# Throw a VPS to it
I fought, why not just throw a VPS to it, and run it 24/7 (or until it hit my limit), I told it to deploy a bunch of things on the server (ofc I did not tell it to setup SSL), and well, it worked, it worked so well, I'm regretting my choices as a software dev. I have not looked a single eye on what code Claude is generating, because It's that good.

Any errors? I just throw it to Claude and continue to scroll Threads. It works? Yeah, another job for Claude. 

Me as a person not writing code anymore is just sad, I still kinda write code, but not as often at the end of 2024, and the start of 2025.
